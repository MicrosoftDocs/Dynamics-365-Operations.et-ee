<span data-ttu-id="dcc33-111">Ajaloolise perioodi lõppu eiratakse, kui prognoosi strateegiaks on määratud **Kopeeri ajalooline nõudlus ümber**.</span><span class="sxs-lookup"><span data-stu-id="dcc33-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="dcc33-112">Süsteem kopeerib väljal **Prognoosi periood** määratud vahemike arvu nõudluse prognoosi, alustades alguskuupäevast, mis on seatud jaotise **Ajalooline periood** väljal **Alguskuupäev**.</span><span class="sxs-lookup"><span data-stu-id="dcc33-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="dcc33-113">Kopeerides ajaloolise nõudluse teatud kuupäevast alates, saavad tootmisplaanijad järgmist kvartalit planeerida kahel viisil:</span><span class="sxs-lookup"><span data-stu-id="dcc33-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

<span data-ttu-id="dcc33-125">Väljal **Alusprognoosi alguskuupäev** - **alates kuupäevast** peab määrama prognoosi vahemiku alguse, näiteks Ameerika Ühendriikides on nädalase prognoosivahemiku puhul alguseks pühapäev.</span><span class="sxs-lookup"><span data-stu-id="dcc33-125">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="dcc33-126">Süsteem kohandab automaatselt välja **Alusprognoosi alguskuupäev** - **alates kuupäevast**, et olla vastavuses prognoosivahemiku algusega.</span><span class="sxs-lookup"><span data-stu-id="dcc33-126">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="dcc33-127">Väljale **Alusprognoosi alguskuupäev** - **alates kuupäevast** saab määrata mineviku kuupäeva.</span><span class="sxs-lookup"><span data-stu-id="dcc33-127">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="dcc33-128">Teisisõnu on võimalik luua nõudluse prognoosi minevikus.</span><span class="sxs-lookup"><span data-stu-id="dcc33-128">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="dcc33-129">See on kasulik, kuna võimaldab kasutajatel prognoosimisteenuse parameetreid kohandada selliselt, et varem loodud statistiline prognoos vastab tegelikule ajaloolisele nõudlusele.</span><span class="sxs-lookup"><span data-stu-id="dcc33-129">This is useful, because it lets users tweak the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="dcc33-130">Seejärel saavad kasutajad neid parameetri sätted tulevase statistilise alusprognoosi loomisel kasutada.</span><span class="sxs-lookup"><span data-stu-id="dcc33-130">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="dcc33-131">Varasemates prognoosimise iteratsioonides käsitsi tehtud korrigeerimised saab automaatselt uuele alusprognoosile rakendada, kui ruut **Kandke käsitsi tehtud
