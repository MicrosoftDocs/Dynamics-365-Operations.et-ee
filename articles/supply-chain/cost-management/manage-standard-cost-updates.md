# <a name="manage-standard-cost-updates"></a><span data-ttu-id="e224f-103">Standardomahinna värskenduste haldus</span><span class="sxs-lookup"><span data-stu-id="e224f-103">Manage standard cost updates</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="e224f-104">Standardomahinna andmete värskendusi saab hallata kahte erinevat lähenemist kasutades: üheversiooniline lähenemine ja kaheversiooniline lähenemine.</span><span class="sxs-lookup"><span data-stu-id="e224f-104">Updates to standard cost data can be managed by using two different approaches - the one-version approach or the two-version approach.</span></span> 

<span data-ttu-id="e224f-105">Üheversiooniline lähenemine kasutab üht kuluversiooni, mis sisaldab kõiki kulukirjeid.</span><span class="sxs-lookup"><span data-stu-id="e224f-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="e224f-106">Need kirjed sisaldavad algseid kulusid ja kõiki kulu värskendusi.</span><span class="sxs-lookup"><span data-stu-id="e224f-106">These records include the original costs and all cost updates.</span></span>

<span data-ttu-id="e224f-107">Kaheversiooniline lähenemine kasutab üht versiooni, mis sisaldab algsete kulude kirjeid, ja teist versiooni, mis sisaldab kõiki kulude värskenduste kirjeid.</span><span class="sxs-lookup"><span data-stu-id="e224f-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="e224f-108">Kaheversioonilise lähenemise peamiseks eeliseks on kulu värskenduste selge kirjeldus ja jälgimine eraldiolevas kuluversioonis ilma, et see mõjutaks originaalkuluversiooni.</span><span class="sxs-lookup"><span data-stu-id="e224f-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="e224f-109">Kaheversioonilist lähenemist saab kasutada mitme järkjärgulise värskenduse tuvastamisel, kus igale järkjärgulisele värskendusele on eraldi järkjärgulisi kulukirjeid sisaldav kuluversioon.</span><span class="sxs-lookup"><span data-stu-id="e224f-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span>

<span data-ttu-id="e224f-128">Taande saab esitada aktiivse kuluna või kuluversioonina 2016-STD, sest mõlemad tuvastavad kuluandmete allika, kui see ei sisaldu versioonis
