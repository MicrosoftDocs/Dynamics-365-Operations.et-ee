# <a name="documents-pending-accounting"></a>Raamatupidamise ootel dokumendid

[!include [banner](../includes/banner.md)]

Selles artiklis kirjeldatakse selle funktsiooni kasutamist lehel **Raamatupidamise ootel dokumendid**.

Microsoft Dynamics 365 Finance 10.0.30 korral on saadaval funktsioon **Lähtedokumentide raamatupidamise raamistiku täiustatud jõudlus**. See funktsioon parendab lähtedokumendi toega dokumentide sisestamise protsesse, alustades vabas vormis arvete sisestamisprotsessist.

Kui see funktsioon on lubatud, käib alammooduli dokumendi sisestamine eraldi raamatupidamise genereerimisest ehk *töölehe sisestuse* protsessist, mis loob pearaamatusse kandmiseks täielikud raamatupidamise üksikasjad. Vabas vormis arve sisestamise protsessis näiteks registreeritakse kliendiarve moodulis **Müügireskontro** enne täieliku raamatupidamise genereerimist. See täiustatud jõudluse funktsioon võimaldab kliendiarveid kiiremini registreerida, kui raamatupidamise loomine peaks hilinema.

Lehel **Raamatupidamise ootel dokumendid** on saadaval järgmised nupud.

- **Loo raamatupidamine** – saate praegu konto genereerimist ootava dokumendi esitada töölehe sisestuseks.
- **Kuva jaotusi** – saate vaadata loendist valitud dokumendi arvestuse jaotusi.
- **Kuva tõrkelogi** – kui raamatupidamise olek osutab tõrgetele, saate vaadata dokumendi tõrketeadete üksikasju. Sama tõrketeate üksikasju saate vaadata ka siis, kui valite dokumendi real lingi **Tõrkelogi**.

Raamatupidamise genereerib protsesside automatiseerimise taustprotsess nimega **Raamatupidamisraamistiku protsessor**. Lisateavet kõigi protsessiautomatiseerimiste häälestamise ja konfigureerimise kohta leiate artiklist [Protsesside automatiseerimine](../../fin-ops-core/dev-itpro/sysadmin/process-automation.md).
